Final Project: Anime Recommender
Student: Justin Miller
School: Georgia Southern University



Description:
System will use a combination of relational and graph database technologies that will compliment each other in order to create a system for identifying patterns in user preference for various shows and recommending TV shows based on perference.
  • The system will have a web front end and login for user interaction.
  • This project will combine concepts learned from every class in this program.
  • Database Design will be extremely important for the login system and wiring of the core application.
  • Web scraping and API mining will be used to pull data from various third parties.
  • Data Warehousing will be used to store the massive amount of data that will be pulled about shows and preferences.
  • AI concepts (data mining) will be used for approaches of mining valuable information from the data (what I should recommend to whom).
  • Web design will be necessary to build a web interface that is easy to use.

URL Pattern

http://csprofessional.net:9000/recommend/$profilename :
This will recommend a set of up to 18 shows based on your ranking profile.
*Note: Profiles correspond to user profiles on http://myanimelist.net

http://csprofessional.net:9000/recommend/$profilename?howMany=# :
This will recommend the specified number of shows based on your ranking profile.

Statistics:

Currently there are 465,494 users, 32,415 shows, and 41,155,169 rankings loaded to base recommendations on.
